# C4 API Tools — Demo Environment

Готовое окружение для демонстрации. Все сервисы запускаются одной командой из предсобранных образов.

## Подготовка

Однократная сборка образов (из корня репозитория):

```bash
cd demo
bash build-images.sh
```

## Запуск

```bash
cd demo
docker compose up -d
```

Первый запуск занимает ~30 секунд:
- PostgreSQL: инициализация БД, создание пользователя `monitoring`
- Nginx: генерация ГОСТ + RSA PKI (CA, серверные и клиентские сертификаты)
- Dashboard: миграции Django, создание пользователя admin

## Доступ

| URL | Описание |
|---|---|
| `https://localhost:8443` | Панель управления (через nginx, ГОСТ + RSA TLS) |

Логин: `admin` / `admin` (настраивается в `.env`)

## Загрузка тестовых конфигураций

Через веб-интерфейс:
1. Откройте **Конфигурация** в боковом меню
2. В блоке **Загрузить JSON-файл конфигурации** выберите файл из `examples/configs/`
3. Нажмите **Импортировать файл**
4. Повторите для каждого файла конфигурации

## Настройка

Переменные окружения — в файле `.env`:

| Переменная | По умолчанию | Описание |
|---|---|---|
| `DASHBOARD_ADMIN_USER` | `admin` | Логин администратора |
| `DASHBOARD_ADMIN_PASSWORD` | `admin` | Пароль администратора |
| `C4_HOST` | `192.168.122.200` | IP сервера Континент 4 |
| `C4_PORT` | `444` | Порт Континент 4 |
| `C4_USER` | `admin` | Пользователь К4 |
| `C4_PASSWORD` | — | Пароль К4 |

## Состав

| Контейнер | Образ | Описание |
|---|---|---|
| `demo-postgresql` | `postgres:16` | PostgreSQL (мониторинг + cus-logs) |
| `demo-nginx-gost` | `c4-nginx-gost` | Nginx с ГОСТ TLS (порты 8443, 8444) |
| `demo-c4-config-exporter` | `c4-config-exporter` | FastAPI — экспорт конфигураций УБ |
| `demo-c4-dashboard` | `c4-dashboard` | Django — панель управления |

## Остановка

```bash
docker compose down
```

Для полной очистки (включая данные PostgreSQL и сертификаты):

```bash
docker compose down -v
```

## Порты

| Порт | Сервис |
|---|---|
| `8443` | nginx → dashboard (ГОСТ + RSA TLS) |
| `8444` | nginx → exporter (ГОСТ mTLS, внутренний) |
